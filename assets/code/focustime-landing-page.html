Core Features of FocusTime

Tailored Focus Sessions

Initiate focused work periods by selecting essential applications and setting a timer. FocusTime then helps you stay on course by managing other applications based on your chosen settings.

  • Application Selection: Choose which apps are vital for your current task.
  • Timed Sessions: Define the length of your focus period.
  • Motivational Quotes: A gentle stream of inspiration in the footer to keep you going.

Thematic Focus Modes

Choose a mode that matches your current discipline level and task demands:

  • Ember (Beginner): "Start the fire." A gentle mode for building momentum. You can stop the session anytime.
  • Forge (Intermediate): "Shape yourself through action." A structured mode where sessions cannot be stopped until the timer ends. Focus apps are minimized at the start to help you dive in.
  • Obsidian (Advanced): "No excuses. Just results." For peak discipline. Sessions are non-stoppable, default to 120 minutes, and focus apps are minimized to ensure deep work.

Smart Application Blocking

Control how FocusTime handles non-essential applications during your sessions:

  • Gentle Mode: Minimizes distracting applications if they come to the foreground.
  • Moderate Mode: Minimizes distracting apps and automatically brings one of your chosen focus apps back to the foreground.
  • Strict Mode: Closes distracting applications if they attempt to take focus.
  • App Whitelisting: Specify applications (e.g., 'explorer.exe', your music player) that should never be blocked, regardless of the mode.

Health & Wellbeing Reminders

Optional, timely notifications to encourage healthy habits during long focus sessions:

  • Eye Strain: Every 20 minutes, a reminder to look away (20-20-20 rule).
  • Movement: Every 30 minutes, a nudge to stand up and stretch.
  • Hydration: Every 45 minutes, a prompt to drink water.

User-Friendly Settings & Operation

  • Minimize to Tray: Keep FocusTime running unobtrusively in the system tray.
  • Notifications: Get informed about session starts, ends, and health reminders.
  • Persistent Settings: Your preferences are saved for future sessions.
  • Performance Tuning: Adjust the blocking check interval if needed.

How FocusTime Can Help, Especially with ADHD:

Individuals with ADHD often face challenges with sustained attention, impulsivity, and time management. FocusTime aims to provide structured support on your Windows PC:

  • Reduces Digital Distractions: By actively managing application access, it minimizes the external digital stimuli that can easily derail focus on your Windows desktop.
  • Encourages Structured Work Intervals: The timer and distinct focus modes promote working in defined blocks, aiding task initiation and completion.
  • Establishes Clear Digital Boundaries: The progressive Focus Modes (Ember, Forge, Obsidian) allow users to choose appropriate restriction levels, helping to build discipline.
  • Provides Visual Time Cues: The prominent countdown timer aids in time perception and offers motivation to stay engaged.
  • Minimizes Task Switching: By making it harder to impulsively switch to non-focus apps, FocusTime helps reduce cognitive friction.

FocusTime is a supportive tool designed to help manage your digital environment on Windows more effectively and cultivate better focus habits.

Build and Run from Source (Windows)

Want to tinker with the code or contribute? Here’s how to get started on Windows:

  1. **Prerequisites:**
    • Windows Operating System.
    • Visual Studio (e.g., 2019, 2022) with the ".NET desktop development" workload installed.
    • Git for Windows (or any Git client).
  2. Clone the repository: 
    git clone https://github.com/yourusername/focustime.git
  3. Navigate to the project directory: 
    cd focustime
  4. Open the solution file (e.g., `FocusTime.sln`) in Visual Studio.
  5. Restore NuGet packages (Visual Studio usually does this automatically, or right-click the solution in Solution Explorer and select "Restore NuGet Packages").
  6. Build the solution (Build > Build Solution or `Ctrl+Shift+B`).
  7. Run the application from Visual Studio (Start > Start Debugging or `F5`). The executable will typically be found in a `bin/Debug` or `bin/Release` subfolder within the project directory.
